Assignment 1

You are going to create a team consisting of x number of members. The team will agree where they are going to store the system. Either locally or on GitHub. Afterwards, you follow the following steps:

  • - Each member studies the system on an individual level, that is by himself/herself. When doing it, the member is expected to make notes about the system quality. Each member is expected create her/his own definition of software system quality without interacting with other members. Let us call it a novice engineer definition. After this course, you are expected to create an expert evaluation of the system.

  • - The task is to evaluate the system from the long-term lifecycle perspective. Is the system quality good enough for evolving and maintaining the system?

  • - There is one problem hindering you from executing this system. Identify it and solve it.

  • - After each member has made her/his own evaluation, the team will meet and the members

will merge their results, agree on common definitions and make a common evaluation.

  • - In case some members join the course and the team in the late stage, the late member is expected to do first his/her own evaluation and then merge her/his results with the team results. At this late stage, the new member does not need to interact with the team. However, the new member’s input should be then considered in Assignment 4. This is Mira’s way of

managing students who join the course in latter stages.

Techs in ExitJS

Let’s consider the whole system has two ares: the front-end, the server, and the database.

  • For the front end, part of it can be reused based on the previous version of this system: CSS, and HTML (extracted from PHP).

  • For the back end, the system migrated from PHP to NodeJS, so the server is completely new. To run the server, we use NodeJS httpServer with the help of Express.

  • For the database, the system uses Postresql.

  4. Enter the name of the database exitjs.
  5. Add user pi with password exitjs567 and all privileges.

SSL credentials

HTTPS is HTTP with a layer of encryption. Before we can enable HTTPS on our Express server, we need to have our SSL certificates ready.

Steps to get free certificate for https
  1. brew install certbot
  2. Use certbot to generate credentials, the credentials would be the first parameter that you will pass to https.createServer().
Two troubles I met

  1. I failed to generate certificates by certbot.

  2. I can not access to the log in letsencrypt.
